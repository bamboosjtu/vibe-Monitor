# 输变电工程数字沙盘系统 - 用户操作说明

**版本**: M1 Round2 当前实现  
**日期**: 2026-05-03

---

## 一、系统简介

输变电工程数字沙盘系统用于展示湖南省范围内输变电工程的站班会作业态势。系统支持：

- 地图展示作业点位（点大小表示人数，点颜色表示风险等级）
- 多维度筛选（作业状态、风险等级、市州、电压等级）
- 详情查看（点击点位查看作业详情）
- 统计分析（作业点数量、人数、风险分布、市州分布）
- **时间轴回溯**（按天查看历史作业态势）
- 图层控制（作业点、杆塔、变电站）

## 三、基本操作

### 3.1 查看地图

- **缩放地图**: 使用鼠标滚轮或地图右上角的 +/- 按钮
- **平移地图**: 按住鼠标左键拖动
- **查看详情**: 点击地图上的作业点圆点、杆塔三角形或变电站五角星

### 3.2 使用筛选

在左侧面板选择筛选条件：

- **作业状态**: 全部 / 作业中 / 作业暂停 / 当日作业完工
- **风险等级**: 一级 / 二级 / 三级 / 四级 / 无风险（可多选）
- **电压等级**: 500kV / 220kV / 110kV / 35kV（可多选）
- **图层控制**: 作业点 / 杆塔 / 变电站

> 提示: 筛选条件变更后，地图和统计会实时更新。

### 3.3 查看详情

1. 点击地图上的任意对象
2. 右侧面板显示对应详情：

作业点包含：
   - 工程名称
   - 风险等级
   - 作业人数
   - 作业状态
   - 电压等级
   - 市州
   - 作业地点
   - 施工单位、监理单位、工作负责人、作业内容等

杆塔包含杆塔编号、所属线路、坐标。变电站包含变电站名称、项目编码、坐标。

## 四、时间轴操作（核心功能）

### 4.1 查看当前日期

顶部标题栏右侧显示：
- **当前日期**: 如 "2026年04月07日"
- **日期位置**: 如 "1 / 4"（表示第1天，共4天）

### 4.2 切换日期

**手动切换**:
- 点击 **"上一天"** 按钮: 切换到前一天的数据
- 点击 **"下一天"** 按钮: 切换到下一天的数据

> 提示: 第一天时"上一天"按钮禁用，最后一天时"下一天"按钮禁用。

**自动播放**:
- 点击 **"播放"** 按钮: 开始按天自动轮播（每1秒切换一天）
- 点击 **"暂停"** 按钮: 停止自动轮播

> 提示: 播放过程中"上一天"/"下一天"按钮禁用，播放到最后一天自动停止。

### 4.3 日期切换时的数据联动

切换日期时，以下数据会自动更新：
- **地图标记**: 显示当前日期的作业点位
- **统计面板**: 重新计算当前日期的统计数据
- **详情面板**: 
  - 若之前选中的记录在新日期存在，则保留选中状态
  - 若之前选中的记录在新日期不存在，则自动清空

> 提示: 筛选条件在日期切换时保持不变。

## 五、异常处理

### 5.1 数据加载失败

如果某日期的数据文件缺失，会显示错误提示：
- 错误信息: "指定日期的数据文件不存在"
- 当前数据保留，不切换到缺失的日期
- 提供 **"重试"** 按钮，可重新尝试加载

### 5.2 日期数据为空

如果某日期没有有效数据：
- 地图清空，不显示标记
- 统计面板显示 "该日期暂无有效数据"
- 详情面板清空

### 5.3 边界提示

- 在第一天点击"上一天": 显示 "已是第一天" 提示
- 在最后一天点击"下一天": 显示 "已是最后一天" 提示
- 在最后一天点击"播放": 显示 "已是最后一天" 提示

## 六、常见问题

**Q: 为什么有些点位看不到？**  
A: 系统会过滤掉坐标无效的记录（经纬度为空、非数字、超出湖南范围或为0）。

**Q: 为什么人数显示和标记大小不一致？**  
A: 人数超过300时，标记大小按300计算（防止标记过大），但详情面板显示实际人数。

**Q: 筛选后为什么详情面板清空了？**  
A: 筛选条件变更后，若之前选中的记录不在筛选结果中，详情面板会自动清空。

**Q: 如何查看历史数据？**  
A: 使用时间轴的"上一天"/"下一天"按钮或"播放"功能切换日期。

**Q: 播放速度可以调整吗？**  
A: 当前版本固定为1秒/天，后续版本可能支持调整。

## 七、数据源模式切换（M0 新增）

M0 阶段支持双数据源模式，默认使用本地静态 JSON 模式。

### 7.1 切换方法

在浏览器控制台（F12）或通过 Zustand DevTools 执行：

```javascript
// 切换到 API 模式（从后端加载数据）
import { useAppStore } from '@/store';
useAppStore.getState().setDataSource('api');

// 切换回本地模式
import { useAppStore } from '@/store';
useAppStore.getState().setDataSource('local');
```

> 注：M0 阶段数据源切换功能在 store 层预留，UI 控件将在后续阶段完善。

### 7.2 模式对比

| 特性 | local 模式（默认） | api 模式 |
|------|-------------------|----------|
| 数据来源 | `/data/daily_meeting/meeting_YYYYMMDD_all.json` | 后端 API `GET /api/map/summary` |
| 适用场景 | 开发调试、离线演示、当前默认演示路径 | 后端联调、导入后数据验证 |
| 性能 | 文件读取，无延迟 | 网络请求，依赖后端响应速度 |
| 数据版本 | 静态快照 | 后端管理，可版本更新 |

> 当前前端 API 客户端默认端口为 `8001`，后端默认端口为 `8000`。使用 API 模式前需要先统一端口配置。
