# 输变电工程数字沙盘系统 - 需求规格说明书（M0-lite）

**版本**: M0-lite v1.0  
**日期**: 2026-04-16  
**当前阶段**: M0-lite（数据底座收口）

**阶段说明**:
- pre-MVP：地图展示与基础交互已具备
- M0-lite：统一数据底座、标识映射、导入覆盖、API 兼容层
- M1：空间主数据与真实骨架地图

---

## 一、项目概述

### 1.1 项目总目标

建设输变电工程数字沙盘系统，形成面向输变电工程的空间对象、项目对象、组织对象与过程对象的一体化数字底座，为后续真实工程骨架地图、项目聚合、组织指挥、事件工单、健康度、战绩履历等模块提供统一的数据基础。

**工程类型定义**:

| 工程类型 | 特点 | 空间表达 |
|----------|------|----------|
| **线路工程** | 线性工程，由杆塔串联形成线路 | 杆塔点位 + 线路几何链 |
| **变电工程** | 单点工程，变电站为独立站点 | 站址坐标点位 |
| **作业对象** | 某一时点的现场作业快照 | 作业点 |

### 1.2 当前阶段目标

M0-lite 不是新增业务功能阶段，而是数据底座收口阶段。

**M0-lite 目标**:
- 将年度目标数据、杆塔数据、meetlist 数据统一纳入项目数据底座
- 建立 `prjCode / singleProjectCode / 空间对象` 的统一标识体系
- 建立导入、覆盖、快照、归档、日志留痕能力
- 为现有 pre-MVP 页面提供 API 兼容层
- 为 M1 空间主数据与真实骨架地图提供稳定数据基础

**M0-lite 不直接新增**:
- 新业务页面
- 新地图业务图层
- 赛季页面
- 组织关系页面
- 事件工单页面
- 健康度、战绩、履历页面

### 1.3 当前阶段不是

M0-lite 不是以下阶段：
- 真实工程骨架地图阶段（属于 M1）
- 项目聚合与地图交互阶段（属于 M2）
- 组织与赛季阶段（属于 M3）
- 事件与工单阶段（属于 M4）

## 二、数据模型

### 2.1 核心管理单元定义

| 单元 | 含义 | 用途 |
|------|------|------|
| **`singleProjectCode`** | 单项工程编码 | 当前系统管理最小单元 |
| **`prjCode`** | 项目编码 | 汇总展示与聚合单元 |

**冻结规则**:
- 管理最小单元使用 `singleProjectCode`
- 汇总展示默认聚合到 `prjCode`
- 年度目标、空间对象、作业点优先挂接到 `singleProjectCode`

### 2.2 数据源定义

| 数据源 | 说明 | 当前阶段用途 |
|--------|------|--------------|
| **年度目标数据** | 按年维护的项目/单项工程目标明细 | 项目与单项工程的计划底座 |
| **杆塔数据** | 线路工程杆塔坐标、序号、上下游关系 | 线路空间骨架底座 |
| **meetlist 数据** | 作业点快照数据，带稳定 `id` | 作业快照底座 |

### 2.3 当前阶段核心字段

#### 2.3.1 年度目标数据

| 字段 | 说明 | 用途 |
|------|------|------|
| `prjCode` | 项目编码 | 项目聚合 |
| `singleProjectCode` | 单项工程编码 | 最小管理单元 |
| `singleProjectTypeName` | 单项工程类型 | 区分线路/变电工程 |
| `buildUnitCode` / `buildUnitName` | 建管单位 | 后续战区归属 |
| `imageProgress` | 年度形象进度 | 当前仅展示，不上图 |
| `jhkgTime` | 计划开工时间 | 上图判定 |
| `sjkgTime` | 实际开工时间 | 上图判定优先字段 |
| `jhtcTime` | 计划投产时间 | 后续阶段使用 |

#### 2.3.2 杆塔数据

| 字段 | 说明 | 用途 |
|------|------|------|
| `singleProjectCode` | 单项工程编码 | 线路归属 |
| `towerNo` | 杆塔编号 | 杆塔唯一识别字段之一 |
| `towerSequenceNo` | 杆塔序号 | 线路几何链主排序依据 |
| `upstreamTowerNo` | 上游塔号 | 拓扑辅助字段 |
| `longitudeEdit` / `latitudeEdit` | 杆塔坐标 | 后续 M1 线路生成 |
| `biddingSectionCode` | 标段编码 | 辅助分析，不作为 M0 主排序依据 |

#### 2.3.3 meetlist 数据

| 字段 | 说明 | 用途 |
|------|------|------|
| `id` | 作业点稳定主键 | 小时快照识别同一对象 |
| `singleProjectCode` | 单项工程编码 | 作业点挂接 |
| 经纬度字段 | 作业点位置 | 作业点快照 |
| 人数/风险/状态字段 | 当前作业快照 | 变化对比 |
| 采集时间字段 | 采集时刻 | 小时快照 |

### 2.4 状态口径

#### 2.4.1 当前冻结口径

- `imageProgress` 当前仅作为展示字段，不上图，不做统一状态机
- 线路对象上图判断基于开工时间，不基于 `imageProgress`
- meetlist 的经纬度变化、人数变化、风险变化、状态变化都算变化

#### 2.4.2 线路对象上图规则

**允许上图**:
1. `sjkgTime` 有值
2. `sjkgTime` 为空，但 `jhkgTime` 已到当前日期

**不上图，只在列表展示**:
1. `jhkgTime` 未到当前日期
2. `jhkgTime` 与 `sjkgTime` 均为空
3. 当前无任何空间对象（无站、无线、无塔）

### 2.5 空间对象边界规则

| 对象 | 当前规则 |
|------|----------|
| **变电站** | 可复用已有稳定唯一主键 |
| **接入关系对象** | 线路接入已有站时允许单独建模 |
| **杆塔** | 站内间隔不算杆塔 |
| **线路** | 一条 `singleProjectCode` 允许拆成多条几何链 |
| **无空间对象单项工程** | 允许只进列表，不上图 |

### 2.6 数据覆盖与快照规则

#### 2.6.1 年度目标数据
- 采用同主键全覆盖业务表全部字段
- 保留历史导入批次与版本快照
- 当前查询默认读取最新生效版本

#### 2.6.2 杆塔数据
- 采用同主键全覆盖业务表全部字段
- 保留历史导入批次与版本快照
- 当前查询默认读取最新生效版本

#### 2.6.3 meetlist 数据
- 以 `id` 识别同一作业点
- 24 小时内保留每小时快照
- 24 小时外按天归档
- 默认读取最新快照
- 对外展示最新值及相较上一采集值的 `+n / -n`

## 三、功能需求

### 3.1 当前阶段功能列表

M0-lite 只允许实现以下能力：

| 序号 | 能力 | 说明 |
|------|------|------|
| 1 | 后端工程初始化 | 建立最小后端底座 |
| 2 | 数据库初始化 | 建立轻量数据底座 |
| 3 | 年度目标数据导入 | 支持全覆盖更新 |
| 4 | 杆塔数据导入 | 支持全覆盖更新 |
| 5 | meetlist 数据导入 | 支持小时快照与归档 |
| 6 | 标识映射 | 建立 `prjCode / singleProjectCode / 空间对象` 映射 |
| 7 | 数据版本留痕 | 导入批次、导入时间、源文件留痕 |
| 8 | API 兼容层 | 现有 pre-MVP 页面可切换读取 API |

### 3.2 当前阶段明确不做

| 序号 | 能力 | 不做理由 |
|------|------|----------|
| 1 | 新地图图层 | 属于 M1 |
| 2 | 线路几何生成 | 属于 M1 |
| 3 | 项目聚合面板 | 属于 M2 |
| 4 | 组织与赛季页面 | 属于 M3 |
| 5 | 事件与工单 | 属于 M4 |
| 6 | 健康度 | 属于 M5 |
| 7 | 战绩与履历 | 属于 M6 |
| 8 | 规则与申诉 | 属于 M7 |

### 3.3 数据导入功能需求

#### 3.3.1 年度目标导入
- 支持按批次导入
- 支持同主键全覆盖更新业务表
- 支持原始表与业务表分离
- 支持错误记录与导入日志

#### 3.3.2 杆塔数据导入
- 支持按批次导入
- 支持同主键全覆盖更新业务表
- 支持异常坐标与缺失字段记录
- 暂不在本阶段生成线路几何链

#### 3.3.3 meetlist 导入
- 支持按批次导入
- 支持小时快照写入
- 支持按天归档
- 支持较上一采集值计算变化量

### 3.4 API 兼容需求

现有 pre-MVP 页面不重写，只增加数据来源切换能力：
- 可读取本地静态 JSON
- 可读取新的 API 数据
- 页面结构、交互、统计口径保持不变

## 四、页面状态与异常交互

### 4.1 M0-lite 页面要求

M0-lite 不新增业务页面，但要支持：
- 后端健康检查
- 导入结果查询
- 地图摘要接口供现有页面读取

### 4.2 异常处理

| 场景 | 处理要求 |
|------|----------|
| 导入文件缺失 | 返回明确错误，不中断已有数据 |
| 文件格式错误 | 返回校验错误，记录导入日志 |
| 同主键覆盖 | 记录覆盖批次与更新时间 |
| meetlist 小时快照异常 | 保留当前成功数据，错误进入日志 |
