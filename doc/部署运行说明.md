## 三、本地开发部署

### 3.1 克隆/解压项目

```bash
# 如果是 Git 仓库
git clone <repository-url>
cd vibe-monitor

# 如果是压缩包
unzip vibe-monitor.zip
cd vibe-monitor
```

### 3.2 安装前端依赖

```bash
cd frontend
npm install
cd ..
```

### 3.3 安装后端依赖

```bash
cd backend
uv sync --dev
cd ..
```

### 3.4 准备数据文件

确保 `./data/daily_meeting/` 目录下存在站班会日期数据文件：

```
data/daily_meeting/
├── meeting_20250101_all.json
├── meeting_20250102_all.json
└── ...                        # 更多日期数据
```

**数据文件格式要求**:
- 文件名格式: `meeting_YYYYMMDD_all.json`
- 每个文件代表一天的历史快照
- 文件内容为 JSON 格式，包含 `list` 数组

### 3.5 初始化数据库

```bash
cd backend
uv run python scripts/db/init_db.py
cd ..
```

### 3.6 配置可用日期列表

系统通过 `frontend/public/data/daily_meeting/index.json` 动态发现可用日期，无需修改代码。

**新增日期的标准步骤**：

1. 将新日期数据文件放入 `data/` 目录，命名为 `meeting_YYYYMMDD_all.json`
2. 运行自动生成脚本：

```bash
cd frontend
npm run generate:data-index
cd ..
```

3. 脚本会自动扫描 `data/daily_meeting/` 目录并更新 `frontend/public/data/daily_meeting/index.json`
4. 刷新页面即可看到新日期

**手动维护（备选）**：如需手动编辑，直接修改 `frontend/public/data/daily_meeting/index.json` 的 `dates` 数组：

```json
{
  "dates": [
    "2026-04-07",
    "2026-04-08",
    "2026-04-09",
    "2026-04-10",
    "2026-04-11"  // 新增日期
  ]
}
```

### 3.7 启动后端服务（可选）

前端默认 `local` 模式，不需要后端即可加载站班会静态 JSON。只有使用导入接口、bootstrap、map summary、skeleton 或切换前端 API 模式时，才需要启动后端。

在项目根目录，使用一个终端启动后端：

```bash
cd backend
uv run python run.py
```

启动成功后，后端默认运行在 `http://localhost:8000`。

### 3.8 启动前端开发服务器

另开一个终端启动前端：

```bash
cd frontend
npm run dev
```

启动成功后，控制台会显示：
```
VITE v5.4.21  ready in XXX ms

➜  Local:   http://localhost:5173/
➜  Network: use --host to expose
```

在浏览器中打开 `http://localhost:5173/` 即可访问。

> 提示：前端默认使用本地静态 JSON 模式（local）。开发/预览环境中，`frontend/vite.config.ts` 的 `serve-repo-data` 插件会把仓库根目录 `data/` 暴露为浏览器路径 `/data/`。如需切换到 API 模式，可在前端 store 中设置 `dataSource` 为 `api`；当前 API 客户端默认端口为 `8001`，后端默认端口为 `8000`，使用前需统一配置。

## 四、生产环境部署

### 4.1 构建生产版本

```bash
cd frontend
npm run build
```

构建完成后，会生成 `frontend/dist/` 目录，包含所有静态文件。

### 4.2 部署方式

#### 方式一：静态文件服务器

将 `frontend/dist/` 目录和仓库根目录 `data/` 部署到同一个 Web 根路径下：

```bash
# 使用 nginx
cp -r frontend/dist/* /var/www/html/
cp -r data/ /var/www/html/data/

# 使用 Apache
cp -r frontend/dist/* /var/www/html/
cp -r data/ /var/www/html/data/

# 使用 Python 简单服务器（测试用）
cd frontend/dist
python -m http.server 8080
```

**重要**：Vite 构建只会复制 `frontend/public/data/` 中的 manifest，不会自动复制仓库根目录 `data/daily_meeting/*.json`。生产环境必须单独把完整 `data/` 目录部署到站点 `/data/` 路径下，否则页面能读取 manifest，但无法加载每日数据。

#### 方式二：Docker 部署

创建 `Dockerfile`:

```dockerfile
FROM nginx:alpine
COPY dist/ /usr/share/nginx/html/
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

创建 `nginx.conf`:

```nginx
server {
    listen 80;
    server_name localhost;
    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /data/ {
        alias /usr/share/nginx/html/data/;
    }
}
```

构建并运行:

```bash
docker build -t vibe-monitor .
docker run -d -p 8080:80 vibe-monitor
```

### 4.3 数据文件部署

确保生产环境 Web 根目录的 `/data/` 包含所有数据文件，且文件权限正确：

```bash
# 检查文件是否存在
ls -la /var/www/html/data/daily_meeting/

# 设置文件权限（如需）
chmod 644 /var/www/html/data/daily_meeting/*.json
```

## 五、常见问题

### 5.1 启动失败

**问题**: `npm run dev` 报错 `Error: Cannot find module 'xxx'`

**解决**:
```bash
# 删除 node_modules 重新安装
cd frontend
rm -rf node_modules package-lock.json
npm install
```

### 5.2 端口被占用

**问题**: `Port 5173 is in use`

**解决**:
```bash
# Vite 会自动尝试其他端口，或手动指定端口
cd frontend
npm run dev -- --port 3000
```

### 5.3 数据文件加载失败

**问题**: 页面显示 "数据加载失败"

**排查**:
1. 检查 `./data/daily_meeting/` 目录是否存在
2. 检查数据文件命名是否符合 `meeting_YYYYMMDD_all.json` 格式
3. 检查文件是否为有效的 JSON 格式
4. 检查浏览器控制台网络请求，确认 `/data/daily_meeting/index.json` 和对应 `meeting_YYYYMMDD_all.json` 均为 200

### 5.4 地图不显示

**问题**: 地图区域空白

**排查**:
1. 检查网络连接（需要访问互联网加载地图底图）
2. 检查浏览器控制台是否有 CORS 错误
3. 尝试更换浏览器或清除缓存

### 5.5 构建失败

**问题**: `npm run build` 报错

**排查**:
```bash
# 先运行 TypeScript 检查
cd frontend
npx tsc --noEmit

# 检查 ESLint
npm run lint
```

## 六、配置说明

### 6.1 地图底图配置

默认使用 OpenStreetMap（开发环境），生产环境建议配置天地图：

编辑 `frontend/constants/map.ts`:

```typescript
// 开发环境
export const OSM_STYLE = 'https://demotiles.maplibre.org/style.json';

// 生产环境（需申请天地图 Key）
export const TDT_STYLE = 'https://tiles.geoapify.com/v1/styles/...';
```

### 6.2 播放速度配置

编辑 `frontend/features/timeline/Timeline.tsx`:

```typescript
// 播放速度配置（毫秒/天）
const PLAY_SPEED_MS = 1000;  // 1秒/天
```

### 6.3 坐标范围配置

编辑 `frontend/lib/dataParser.ts`:

```typescript
// 湖南省大致范围
const HUNAN_BOUNDS = {
  lngMin: 108.6,
  lngMax: 114.3,
  latMin: 24.6,
  latMax: 30.2,
```

## 七、维护说明

### 7.1 更新数据文件

1. 准备新的日期数据文件
2. 按命名规范放入 `./data/` 目录：`meeting_YYYYMMDD_all.json`
3. 进入 `frontend/`，运行自动生成脚本更新 `frontend/public/data/daily_meeting/index.json`：

```bash
cd frontend
npm run generate:data-index
```

4. 重新构建部署（生产环境）：

```bash
npm run build
```

### 7.2 日志查看

开发环境日志在浏览器控制台查看：
- Windows: `F12` 或 `Ctrl+Shift+J`
- Mac: `Cmd+Option+J`

### 7.3 性能监控

使用浏览器 DevTools 监控性能：
- Performance 面板: 分析加载和运行时性能
- Memory 面板: 检查内存泄漏
- Network 面板: 检查资源加载
