# 输变电工程数字沙盘系统 - 需求规格说明书

**版本**: MVP v1.1  
**日期**: 2026-04-11  
**当前阶段**: MVP 第二轮（时间轴与历史回溯 - 按天版本）

**历史版本**:
- MVP v1.0 (2026-04-05): 当前作业上图闭环
- MVP v1.1 (2026-04-11): 时间轴与历史回溯（按天版本）

---

## 一、项目概述

### 1.1 项目总目标

建设输变电工程数字沙盘系统，支持线路工程、变电工程、作业态势等空间可视化，为工程管理提供态势感知能力。

**工程类型定义**:

| 工程类型 | 特点 | 空间表达 |
|----------|------|----------|
| **线路工程** | 线性工程，由杆塔串联形成线路 | 杆塔点位 + 线路轨迹 |
| **变电工程** | 单点工程，变电站为独立站点 | 站址坐标点位 |

### 1.2 当前阶段目标

**第一轮已完成**: 当前作业上图闭环 ✓

MVP 第一轮已完成以下基础能力：
- 湖南省范围地图展示
- 站班会点位上图（点大小映射人数，点颜色映射风险等级）
- 多维筛选（状态、风险等级、市州、电压等级）
- 详情面板展示
- 统计面板展示
- 页面状态处理

**当前第二轮目标**: 时间轴与历史回溯（按天版本）

在保持第一轮所有功能的基础上，新增：
- 支持按天切换历史数据
- 时间轴组件提供日期导航
- 地图、筛选、详情、统计随日期联动
- 播放/暂停控制（按天粒度）

**明确不做**:
- 按小时切片的时间轴
- 历史轨迹回放（仅支持按天切换快照）
- 复杂动画效果

## 二、数据模型

### 2.1 空间对象定义

| 空间对象 | 业务含义 | 坐标来源 | 本轮展示 |
|----------|----------|----------|----------|
| **站班会点位** | 站班会召开位置 / 当前作业聚集点 | `toolBoxTalkLongitude/Latitude` | **是** |
| **变电工程点位** | 变电站站址坐标 | 独立站址坐标数据（字段细节待后续补充） | **否**（不单独渲染独立图层） |
| **杆塔点位** | 线路工程杆塔位置 | 杆塔信息.csv | **否** |

**统一点位语义**: 地图只渲染一个点位图层，即**站班会点位图层**。变电工程点位概念保留，但不单独渲染独立图层；若记录属于变电工程，暂通过站班会点位完成地图表达。后续再引入独立站址点图层。

### 2.2 关联规则（已确认）

`singleProjectCode` 与杆塔信息的关联规则已确认，关联逻辑如下：
- 线路工程通过 `singleProjectCode` 匹配杆塔序列
- 当前问题不是关联规则不成立，而是尚未抓取到全部关联杆塔信息
- **因此不展示杆塔和线路轨迹**，待杆塔数据抓取稳定后，再进入线路轨迹展示

### 2.3 核心数据字段

| 字段 | 说明 | 用途 |
|------|------|------|
| `toolBoxTalkLongitude/Latitude` | 站班会召开地点坐标 | 地图标记定位 |
| `currentConstrHeadcount` | 当前作业人数 | 标记大小驱动 |
| `reAssessmentRiskLevel` | 复评风险等级（1-4） | 标记颜色驱动 |
| `currentConstructionStatus` | 当前施工状态 | 筛选条件 |
| `voltageLevel` | 电压等级 | 筛选条件（AC00351=35kV, AC01101=110kV, AC02201=220kV, AC05001=500kV） |
| `buildUnitName` | 建管单位（市州） | 筛选条件 |
| `workStartTime` | 作业开始时间 | 详情展示 |
| `prjName` | 项目名称 | 详情展示 |

### 2.4 状态口径

**当前状态口径严格与已逆向确认的真实页面保持一致**:

| 状态值 | 含义 |
|--------|------|
| **全部** | 不限制状态 |
| **作业中** | 当前正在进行的作业 |
| **作业暂停** | 已暂停的作业 |
| **当日作业完工** | 当日已完成的作业 |

**注意**: 不再使用"未开始 / 进行中 / 已结束"等自定义口径。

### 2.5 字段口径说明

**项目名称**: 优先使用 `prjName`；若后续验证发现真实业务标题由多字段拼接，则按真实标题规则调整。

**风险等级**: 若 `reAssessmentRiskLevel` 为空、缺失或不在1-4范围内，则按"未知风险"样式处理。

**市州口径**: 当前MVP中，"市州"筛选暂按 `buildUnitName` 口径组织；若后续发现其与行政区划不完全一致，再单独调整。

### 2.6 数据清洗与异常处理规则

对异常数据采用"容错展示，降级渲染"策略，确保单条数据异常不影响整体页面可用性。

#### 2.6.1 经纬度异常处理

| 异常场景 | 判定规则 | 处理策略 |
|----------|----------|----------|
| 经纬度为空 | `toolBoxTalkLongitude` 或 `toolBoxTalkLatitude` 为 `null`、`undefined`、空字符串 | **过滤掉该记录**，不渲染在地图上，不计入统计 |
| 经纬度格式非法 | 无法解析为有效数字 | 同上，过滤掉 |
| 经纬度超范围 | 经度不在 [108, 114] 范围或纬度不在 [24, 30] 范围（湖南省大致范围） | 同上，过滤掉 |
| 经纬度为 0 | 经度或纬度等于 0 | 同上，过滤掉（视为未录入坐标） |

**说明**: 不尝试智能修复或猜测坐标，直接过滤。过滤后的记录数量应在浏览器控制台输出日志供调试。

#### 2.6.2 当前作业人数异常处理

| 异常场景 | 判定规则 | 处理策略 |
|----------|----------|----------|
| 人数为空 | `currentConstrHeadcount` 为 `null`、`undefined`、空字符串 | 视为 **0 人**，点位半径取最小值（5px），详情显示"0人" |
| 人数为负数 | 数值小于 0 | 视为 **0 人**（负数视为脏数据，丢弃处理） |
| 人数为非数字 | 无法解析为数字 | 视为 **0 人** |
| 人数超过 300 | 数值大于 300 | 半径按 300 计算（最大值），但**详情面板如实显示原始人数** |

**统一规则说明**: 所有异常人数（空/非数字/负数）统一视为 0 人。负数取绝对值可能掩盖数据质量问题（如-50人变成50人），故采用丢弃策略。

#### 2.6.3 施工状态异常处理

| 异常场景 | 判定规则 | 处理策略 |
|----------|----------|----------|
| 状态值为空 | `currentConstructionStatus` 为 `null`、`undefined`、空字符串 | **归类为"未知状态"**，默认不命中任何筛选条件，仅在"全部"状态下显示 |
| 状态值为脏值 | 不在已定义的 4 个状态值范围内 | 同上，归类为"未知状态" |

#### 2.6.4 风险等级异常处理

| 异常场景 | 判定规则 | 处理策略 |
|----------|----------|----------|
| 风险等级为空 | `reAssessmentRiskLevel` 为 `null`、`undefined`、空字符串 | **命中"未知风险"样式**（灰色 #9CA3AF） |
| 风险等级非法 | 不在 [1, 2, 3, 4] 范围内 | 同上，命中"未知风险"样式 |

**未知风险纳入统计规则**: 未知风险记录计入"当日作业点总数"，但在"风险分布"统计中单独作为"未知"类别展示，不计入 1-4 级的数量。

#### 2.6.5 详情字段为空处理

| 字段 | 空值展示策略 |
|------|--------------|
| `prjName` | 显示为 "--" |
| `ticketNo` | 显示为 "--" |
| `currentConstrHeadcount` | 显示为 "0人" |
| `reAssessmentRiskLevel` | 显示为 "未知"（灰色字体） |
| `workStartTime` | 显示为 "--" |
| `workSiteName` | 显示为 "--" |
| `leaderName` | 显示为 "--" |
| `safeStringNames` | 显示为 "--" |
| `toolBoxTalkAddress` | 显示为 "--" |

#### 2.6.6 同坐标多记录处理

**决策**: **允许重叠**，不做聚合或散开处理。

- 多条记录坐标相同时，在地图上会完全重叠显示
- 用户点击该坐标时，**只展示最上层（数据顺序中最后一条）记录的详情**
- **界面提示**: 当用户点击存在多条记录重叠的坐标时，详情面板顶部显示提示文案："该位置存在多条作业记录，当前仅展示其中一条"
- 重叠问题在后续阶段通过聚合或散开策略优化

**重要说明**: 
- 该策略仅为**临时交互简化**，不代表业务上该坐标只有一条记录
- 用户应被明确告知存在多条记录，避免产生误解

## 三、功能需求

### 3.1 功能分期

#### MVP 第一轮（已完成）

**目标**: 当前作业上图闭环

- [x] 地图底图展示（湖南省范围）
- [x] 当前作业点上图（站班会位置）
- [x] 点大小受人数影响（`currentConstrHeadcount`）
- [x] 点颜色受风险等级影响（`reAssessmentRiskLevel`）
- [x] 基础筛选：状态 / 风险等级 / 市州 / 电压等级
- [x] 点击点位显示基础详情（基于列表页稳定字段）
- [x] 基础统计信息展示

**明确不包含**:
- 线路轨迹展示
- 时间轴功能
- 数据刷新机制
- 详情页深字段

#### MVP 第二轮（当前）

**目标**: 时间轴与历史回溯（按天版本）

**功能列表**:
- [ ] 时间轴组件 - 展示当前日期，提供日期切换
- [ ] 多日期静态数据加载 - 按日期加载对应 JSON 文件
- [ ] 日期切换联动 - 地图、筛选、详情、统计随日期更新
- [ ] 播放/暂停控制 - 按天自动轮播

**时间粒度**:
- 按天切片（YYYY-MM-DD）
- **不按小时切片**（本轮仅按天实现）

**数据来源**:
- 数据目录: `./data/`
- 文件命名: `meeting_YYYYMMDD_all.json`
- 示例: `meeting_20260407_all.json`、`meeting_20260408_all.json`
- 每个 JSON 文件代表一天的历史快照

**时间轴交互**:
- 日期显示: 当前选中日期（YYYY年MM月DD日格式）
- 日期切换: 上一天 / 下一天按钮
- 位置指示: 显示当前是第几天/总共几天
- 边界控制: 第一天禁用"上一天"，最后一天禁用"下一天"
- 加载状态: 切换日期时显示加载指示器

**日期切换行为**:
- 筛选条件: 保持不变（用户可跨日期保持筛选视角）
- 选中项: 若选中项在新日期不存在，自动清空详情面板
- 统计口径: 各日期保持一致

**异常处理**:
- 日期文件缺失: 走现有错误状态体系，显示"数据加载失败"
- 当天数据为空: 显示"暂无有效数据"状态
- 日期越界: 第一天/最后一天按钮禁用
- 加载失败: 保留当前数据，显示错误提示

**注意**: 不承诺复杂动画和精确过程回放。

### 3.2 地图展示

#### 3.2.1 底图要求

- 范围: 湖南省全境（约东经108°-114°，北纬24°-30°）
- 支持缩放和平移

#### 3.2.2 标记设计

**只展示站班会点位**:

| 视觉属性 | 驱动字段 | 规则 |
|----------|----------|------|
| **位置** | `toolBoxTalkLongitude/Latitude` | 直接使用 |

| **颜色** | `reAssessmentRiskLevel` | 4级=#FF3B30(红), 3级=#FF9500(橙), 2级=#FFD700(黄), 1级=#34C759(绿)，未知风险=#9CA3AF(灰) |

### 3.3 筛选面板（左侧）

**筛选条件**:

| 筛选项 | 选项 | 交互类型 |
|--------|------|----------|
| 作业状态 | 全部 / 作业中 / 作业暂停 / 当日作业完工 | 单选 |
| 风险等级 | 一级 / 二级 / 三级 / 四级 / 无 | 多选 |
| 市州 | 长沙 / 岳阳 / 邵阳 / 永州 / 常德 / 衡阳 / 株洲 / 湘潭 / 益阳 / 郴州 / 怀化 / 娄底 / 湘西 / 张家界 / 省建设公司 | 多选（暂删） |
| 电压等级 | 35kV / 110kV / 220kV / 500kV | 多选 |

#### 3.3.1 市州筛选项来源决策

**决策**: 采用**固定业务枚举列表**。

**枚举值**（14个市州 + 1个省直属）:
```
长沙、岳阳、邵阳、永州、常德、衡阳、株洲、湘潭、益阳、郴州、怀化、娄底、湘西、张家界、省建设公司
```

**说明**: "省建设公司"代表省直属项目，与14个地市并列，共同构成建管单位的完整业务枚举。

**选择固定枚举的理由**:
1. 业务上湖南省市州范围稳定，不会频繁变动
2. 固定枚举可确保筛选面板展示顺序一致，用户体验可控
3. 动态生成可能导致数据缺失时市州列表不完整（如某日无某市州数据）
4. 便于前端实现全选/反选功能

**异常值处理规则**:

| 场景 | 处理策略 |
|------|----------|
| `buildUnitName` 不在固定业务枚举中 | 该记录**仍显示在地图上**，但在"市州分布"统计中归入"其他"类别 |
| `buildUnitName` 为空 | 同上，归入"其他" |
| 筛选某市州时 | 只命中完全匹配的记录；值为"其他"的记录只在选择"全部"时显示 |

**说明**: "其他"类别不参与市州 Top5 排名，仅在完整列表中展示。

**响应**: 筛选条件变更后，前端基于当前已加载的数据立即更新地图标记和统计展示。

## 四、时间轴功能需求（第二轮新增）

### 4.1 功能概述

时间轴功能支持按天切换历史数据，查看不同日期的作业态势。

### 4.2 核心功能

#### 4.2.1 当前日期展示

- 在页面顶部显示当前选中的日期
- 格式：YYYY年MM月DD日
- 同时显示当前是第几天/总共几天（如：1 / 4）

#### 4.2.2 上一天 / 下一天

- 提供"上一天"按钮，点击切换到前一天数据
- 提供"下一天"按钮，点击切换到下一天数据
- 第一天禁用"上一天"按钮
- 最后一天禁用"下一天"按钮

#### 4.2.3 播放 / 暂停

- 提供"播放"按钮，点击后按天自动轮播
- 提供"暂停"按钮，点击后停止轮播
- 播放速度：默认 1 天/秒（可配置）
- 播放到最后一天自动停止

#### 4.2.4 默认加载最新一天

- 页面初始化时，自动加载可用日期中最新的那一天
- 可用日期列表预定义或自动发现

#### 4.2.5 日期切换联动

切换日期时，以下模块必须同步更新：

| 模块 | 联动行为 |
|------|----------|
| 地图 | 重新渲染当前日期的点位标记 |
| 统计面板 | 重新计算当前日期的统计数据 |
| 详情面板 | 若选中项在新日期存在则保留，不存在则清空 |
| 筛选结果 | 基于新数据重新应用当前筛选条件 |

### 4.3 数据加载规则

- 数据目录：`./data/`
- 文件命名：`meeting_YYYYMMDD_all.json`
- 每个文件代表一天作业快照
- 前端根据当前日期加载对应文件

## 五、时间轴异常交互（第二轮新增）

### 5.1 日期文件缺失

| 场景 | 交互要求 |
|------|----------|
| 目标日期文件不存在 | 显示错误提示"指定日期的数据文件不存在" |
| 错误处理 | 保留当前日期数据，不切换 |
| 用户提示 | 提供"重试"或"返回"选项 |

### 5.2 日期数据为空

| 场景 | 交互要求 |
|------|----------|
| 目标日期文件存在但无有效数据 | 地图清空标记 |

## 六、页面状态与异常交互

### 6.1 加载状态（Loading）

| 场景 | 交互要求 |
|------|----------|
| 页面初始化 | 显示全屏 Loading 遮罩，文案"数据加载中..." |
| 日期切换 | 显示加载指示器，时间轴按钮禁用 |
| 数据加载完成 | Loading 消失，地图和面板淡入显示（过渡动画 300ms） |

**技术细节**: Loading 期间地图容器可预先渲染底图，但标记层等待数据就绪后再渲染。

### 6.2 数据加载失败

| 场景 | 交互要求 |
|------|----------|
| JSON 文件读取失败 | 显示错误提示面板，文案"数据加载失败，请刷新页面重试"，提供"刷新"按钮 |
| 数据解析异常 | 同上 |
| 日期文件缺失（第二轮） | 显示错误提示，文案"指定日期的数据文件不存在"，保留当前日期数据 |

**容错要求**: 即使部分数据解析失败，也应展示已成功解析的部分，不完全白屏。

### 6.3 空数据状态

| 场景 | 交互要求 |
|------|----------|
| 原始数据为空数组 | 地图正常展示底图，统计面板显示"暂无数据"，详情面板显示"请点击地图标记查看详情"的占位文案 |
| 所有记录因坐标非法被过滤 | 同上，同时在浏览器控制台输出警告日志 |
| 某日期无有效数据（第二轮） | 地图清空标记，统计面板显示"该日期暂无有效数据"，详情面板清空 |

### 6.4 筛选后无结果

| 场景 | 交互要求 |
|------|----------|
| 筛选条件组合无匹配数据 | 地图清空标记，统计面板显示"当前筛选条件下无数据"，详情面板清空 |

### 6.5 日期越界处理（第二轮）

| 场景 | 交互要求 |
|------|----------|
| 当前为第一天，点击"上一天" | 按钮禁用，Tooltip 提示"已是第一天" |
| 当前为最后一天，点击"下一天" | 按钮禁用，Tooltip 提示"已是最后一天" |
| 播放到边界 | 自动停止播放 |

### 6.6 当前选中项被筛掉后的处理

**决策**: 筛选条件变更后，**清空当前选中状态**。

- 详情面板恢复到默认占位文案："请点击地图标记查看详情"
- 地图无高亮标记
- rationale: 避免用户困惑（为什么详情展示的内容在地图上找不到对应标记）

**第二轮新增 - 日期切换时的选中项处理**:
- 若选中项在新日期数据中仍然存在，**保留选中状态**
- 若选中项在新日期数据中不存在，**清空详情面板**
- rationale: 避免展示过期数据，保持数据一致性

**后续优化方向**（非本轮）: 可保留选中并提示"当前选中项不在筛选结果中，是否清空筛选条件查看"。

## 八、统计口径定义

所有统计指标均基于**当前筛选结果**实时计算，口径如下：

**重要说明**：时间轴切换不改变统计口径，每个日期的数据仍按原有规则清洗与统计。

### 8.1 当日作业点总数

**定义**: 当前筛选条件下，拥有**合法坐标**（经度、纬度均有效且在范围内）的记录数量。

**计算公式**:
```
当日作业点总数 = 筛选结果中 (toolBoxTalkLongitude 有效 AND toolBoxTalkLatitude 有效) 的记录数
```

**明确排除**:
- 经纬度为空的记录
- 经纬度格式非法的记录
- 经纬度超范围的记录

### 8.2 当前作业总人数

**定义**: 当前筛选条件下，**拥有合法坐标的记录**的 `currentConstrHeadcount` 之和。

**数据归一化规则**:
```
对于每条记录的人数值，生成两个用途的值：

1. 半径映射值（用于地图标记大小）:
   - 空/非数字/负数 → 0
   - 0-300 → 原值
   - >300 → 300（截断）

2. 统计/详情值（用于统计和详情展示）:
   - 空/非数字/负数 → 0
   - 其他 → 原值（如实显示，不截断）
```

**统计范围说明**: 作业总人数与作业点总数保持同一统计范围（仅含合法坐标记录），确保"所见即所得"。坐标非法的记录不参与任何统计指标。

### 8.3 风险分布

**统计维度**: 按**记录数**统计（不按人数）。

**分类**:
- 一级风险（reAssessmentRiskLevel = 1）
- 二级风险（reAssessmentRiskLevel = 2）
- 三级风险（reAssessmentRiskLevel = 3）
- 四级风险（reAssessmentRiskLevel = 4）
- 未知风险（reAssessmentRiskLevel 为空或非法）

**注意**: 未知风险单独作为一个类别统计，饼图/环形图需包含该类别的展示。

### 8.4 市州分布 Top5

**统计维度**: 按**记录数**排序（不按人数）。

**排序规则**:
1. 按记录数降序排列
2. 记录数相同时，按市州名称字母序排列

**Top5 选取**: 取排名前 5 的市州，**不包含"其他"类别**。

**示例**:
```
长沙: 85条
岳阳: 42条
邵阳: 38条
永州: 35条
常德: 28条
衡阳: 25条 (第6，不展示)
其他: 15条 (不参与排名)
```

### 8.5 数据一致性校验

**校验规则**: 在任意筛选条件下，以下等式应成立：
```
地图标记数量 == 当日作业点总数
```

**不一致时的处理**: 以地图标记数量为准，在浏览器控制台输出警告日志供调试。
