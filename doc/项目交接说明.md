## 一、快速启动

### 前端本地模式（默认演示路径）

```bash
cd frontend
npm install
npm run dev
```

打开 `http://localhost:5173/`。

前端默认 `dataSource: 'local'`，不需要先启动后端。Vite 配置中的 `serve-repo-data` 插件会把仓库根目录的 `data/` 暴露为浏览器里的 `/data/`：

| 浏览器请求 | 实际读取 |
|-----------|----------|
| `/data/daily_meeting/index.json` | 优先 `frontend/public/data/daily_meeting/index.json` |
| `/data/daily_meeting/meeting_YYYYMMDD_all.json` | `data/daily_meeting/meeting_YYYYMMDD_all.json` |

### 后端

```bash
cd backend
uv sync --dev
uv run python scripts/db/align_db.py
uv run python run.py
```

后端默认运行在 `http://localhost:8000`，Swagger: `http://localhost:8000/docs`。

## 三、当前数据流

### 前端 local 模式

1. `useDataLoader()` 在页面初始状态读取 `/data/daily_meeting/index.json`。
2. `dateDataLoader.ts` 取 manifest 中最后一个日期作为默认日期。
3. 按日期请求 `/data/daily_meeting/meeting_YYYYMMDD_all.json`。
4. `dataParser.ts` 清洗并归一化原始站班会数据。
5. Zustand store 保存 `normalizedData`、`filteredData`、时间轴状态、选中对象和图层可见性。
6. 地图、详情、统计、时间轴从 store 读取状态并联动。

### 后端 API 模式

前端 store 支持 `dataSource: 'api'`，会同时读取：

- `/data/daily_meeting/index.json`
- `GET /api/map/summary`

但当前 `frontend/api/config.ts` 默认 API baseUrl 是 `http://localhost:8001`，而 `backend/run.py` 默认启动 `http://localhost:8000`。接手者如果要使用 API 模式，需要先统一端口配置。

## 四、当前功能边界

已实现：

- 按天时间轴，默认加载 manifest 中最新日期
- 作业点地图渲染，点大小映射人数，颜色映射风险等级
- 作业状态、风险等级、电压等级筛选
- 统计面板、详情面板、图层控制
- 后端 SQLite 数据底座、数据导入接口、bootstrap、map summary、skeleton API
- 杆塔和变电站骨架图层；线路数据由后端返回但前端暂不渲染

当前限制：

- 线路数据由后端 skeleton 返回，但前端暂不渲染线路图层
- `year_progress_formation` 缺少稳定 `singleProjectCode`，当前仍有 unresolved 记录
- 前端地图组件内保留了 M1R2 调试覆盖层和大量 console 日志
- 前端 API 模式端口配置与后端默认端口不一致
- 后端 `tests/` 目前没有正式 pytest 用例
- 生产静态部署时，`data/daily_meeting/*.json` 不会自动进入 `frontend/dist`，需要单独部署到 Web 根目录的 `/data/daily_meeting/`
