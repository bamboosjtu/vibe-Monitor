# 运行与联调

## 1. 启动 DataHub

```powershell
cd D:\vibe-coding\vibe-workspace\vibe-DataCollectorHub
uv run run.py
```

确认 health：

```powershell
Invoke-RestMethod -Method Get -Uri "http://127.0.0.1:8000/health" |
ConvertTo-Json -Depth 10
```

## 2. 启动 Monitor backend 和 frontend

### Monitor backend mode

```powershell
cd D:\vibe-coding\vibe-workspace\vibe-Monitor\backend
uv run python run.py
```

```powershell
cd D:\vibe-coding\vibe-workspace\vibe-Monitor\frontend

$env:VITE_DATA_SOURCE = "monitor_backend"
$env:VITE_MONITOR_BACKEND_URL = "http://localhost:8001"
npm run dev
```

打开：

```text
http://localhost:5173
```

### Local fallback

```powershell
cd D:\vibe-coding\vibe-workspace\vibe-Monitor\frontend

$env:VITE_DATA_SOURCE = "local"
npm run dev
```

## 3. 手动验收

Monitor backend API：

```powershell
Invoke-RestMethod -Method Get -Uri "http://localhost:8001/api/map/dates" |
ConvertTo-Json -Depth 10

Invoke-RestMethod -Method Get -Uri "http://localhost:8001/api/map/summary" |
ConvertTo-Json -Depth 10

Invoke-RestMethod -Method Get -Uri "http://localhost:8001/api/map/skeleton" |
ConvertTo-Json -Depth 10
```

Monitor 页面：

- 时间轴显示 backend dates。
- 地图显示 work points。
- 地图显示 towers。
- 地图显示 stations。
- 切换日期后 work points 更新。
- skeleton 图层不因日期切换重复请求。
- 详情面板不展示 `raw`。
- `line_section` 详情通过项目域接口读取，不直接作为当前地图图层。

## 5. 常见问题

### 页面没有数据

先查 Monitor backend：

```powershell
Invoke-RestMethod -Method Get -Uri "http://localhost:8001/api/map/dates" |
ConvertTo-Json -Depth 10
```

如果没有日期数据，说明 DataHub 尚无 canonical work_point，需要先从 downloader 同步并运行 processing。

### 浏览器 CORS 错误

Monitor backend 当前应允许 frontend 本地开发访问：

```text
http://localhost:5173
http://127.0.0.1:5173
http://localhost:4173
http://127.0.0.1:4173
```

如果 Vite 端口变化，需要同步调整 Monitor backend CORS 配置或固定 Vite 端口。

### backend 与 DataHub 端口

- DataHub 默认端口：`8000`
- backend 默认端口：`8001`

两者不会冲突。如需同时运行，请确认各自端口配置。
