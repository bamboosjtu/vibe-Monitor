## 四、测试环境建议

### 4.1 单元测试框架

当前前端已配置 Vitest、Testing Library、jest-dom 和 jsdom。常用命令：

```bash
cd frontend
npm run test:run
npm run test
npm run test:coverage
```

### 4.2 测试脚本配置

`frontend/package.json` 当前脚本：

```json
{
  "scripts": {
    "test": "vitest",
    "test:run": "vitest --run",
    "test:ui": "vitest --ui",
    "test:coverage": "vitest --coverage"
  }
}
```

### 4.3 测试文件组织

```
frontend/
├── lib/
│   ├── dateDataLoader.test.ts
│   ├── dataParser.ts
│   ├── filter.ts
│   ├── stats.ts
│   └── dateDataLoader.ts
├── store/
│   ├── index.ts
│   └── index.test.ts
├── features/timeline/
│   ├── Timeline.tsx
│   └── Timeline.test.tsx

    └── generate-data-index.test.js
```

## 七、已知限制

当前项目已配置前端 Vitest 测试框架，已有覆盖：

- `frontend/scripts/generate-data-index.test.js`
- `frontend/lib/dateDataLoader.test.ts`
- `frontend/store/index.test.ts`
- `frontend/features/timeline/Timeline.test.tsx`

后端已创建 `backend/tests/` 作为 pytest 目录，但当前尚无正式后端自动化测试。`backend/scripts/diagnostics/` 下的文件是人工诊断脚本，不应当视为 pytest 测试。
