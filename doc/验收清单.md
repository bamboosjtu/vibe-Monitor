## 五、快速测试指南

### 5.1 正常流程测试

1. 打开页面，确认默认加载最新日期
2. 点击"上一天"，观察数据变化
3. 点击"下一天"，回到最新日期
4. 选中一个记录，切换日期，观察选中态处理

### 5.2 边界测试

1. 连续点击"上一天"直到第一天，确认按钮禁用
2. 连续点击"下一天"直到最后一天，确认按钮禁用
3. 在边界点击禁用按钮，确认提示出现

### 5.3 异常测试

1. 删除 `./data/daily_meeting/meeting_20260408_all.json`
2. 尝试切换到 2026-04-08
3. 确认显示错误提示，保留当前日期数据
4. 点击"重试"，确认尝试重新加载
5. 恢复被删除的文件

### 5.4 一致性测试

1. 记录当前日期的"当日作业点总数"
2. 手动数地图上的标记数量
3. 确认两者相等
4. 切换筛选条件，重复验证

### 5.5 播放控制测试

1. 点击"播放"按钮，观察是否开始自动切换
2. 观察播放速度是否为 1 天/秒
3. 点击"暂停"按钮，观察是否停止
4. 播放至最后一天，观察是否自动停止
5. 在最后一天点击"播放"，观察是否提示无法播放
6. 播放过程中删除即将加载的文件，观察是否自动停止并显示错误

### 7.8 当前已知限制

#### year_progress_formation 主键问题

正式源数据 `raw_data[*]` 中**不存在** `singleProjectCode` 字段，只有列显示值 `code` 和 `name`。

**当前采用 unresolved 落库策略**：

| 字段 | 值 | 说明 |
|------|-----|------|
| `single_project_code` | `NULL` | 保留字段，后续映射回填 |
| `source_code` | 源数据 `code` | 源数据中的列显示值 |
| `source_name` | 源数据 `name` | 源数据中的项目名称 |
| `key_type` | `"source_code_fallback"` | 主键来源标记 |
| `is_resolved` | `false` | 未解析标记 |

**禁止事项**：
- ❌ 禁止后续 agent 把 `code` 冒充为 `singleProjectCode`
- ❌ 禁止伪造 `singleProjectCode`
- ❌ 禁止在未建立映射关系的情况下设置 `is_resolved = true`

**后续要求**：
- 必须通过 `prjCode / singleProjectCode / 空间对象` 映射关系回填 `single_project_code`
- 回填后方可设置 `is_resolved = true`、`key_type = "resolved"`

#### map_summary 状态映射

部分 `work_status` 值为原始编码（如 `"03"`），尚未映射为中文状态标签。此为 M0 范围未实现功能，不是 silently fallback。后续 M1 可通过状态枚举映射表解决。
