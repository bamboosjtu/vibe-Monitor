**版本**: v1.0

**状态**: 策划阶段，等待数据sample

## 一、项目背景与愿景

### 1.1 背景

输变电工程数字沙盘系统已完成MVP第二轮开发，具备：
- 作业点态势可视化（站班会点位上图）
- 时间轴与历史回溯（按天版本）
- 筛选、统计、详情面板联动

### 1.2 愿景

将数字沙盘升级为**"基建online"**——一个融合《三国志》《文明》等4X策略游戏体验的严肃游戏（Serious Game），用游戏化的交互方式呈现真实的工程管理数据。

**核心理念**：
- 不是传统意义上的"游戏"，而是**数据的可玩性呈现**
- 所有游戏机制必须**基于真实业务逻辑**
- 目标是**提升管理趣味性和数据洞察效率**

### 1.3 设计原则

1. **业务真实性优先**：所有游戏机制必须有业务对应
2. **数据驱动**：所有数值来源于真实数据源
3. **可视化增强**：用游戏化视觉提升数据可读性
4. **正向激励**：通过成就系统激发管理积极性

## 二、核心游戏机制

### 2.1 双体系架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                         指挥体系（Command）                          │
│                      建管单位 = 指挥部/军师府                         │
├─────────────────────────────────────────────────────────────────────┤
│  角色定位：运筹帷幄，决胜千里                                        │
│  核心职能：项目策划、资源协调、进度督导、质量把控                      │
│  关键特征：                                                          │
│  ├── 无直接作战单位（无士兵）                                        │
│  ├── 可调度多个作战单位（可任命多个施工单位）                        │
│  ├── 通过"军令"影响战局                                             │
│  └── 评价基于项目整体结果                                            │
└─────────────────────────────────────────────────────────────────────┘
                                    ↓ 任命/指挥
┌─────────────────────────────────────────────────────────────────────┐
│                         作战体系（Combat）                           │
│                      施工单位 = 军团/野战军                           │
├─────────────────────────────────────────────────────────────────────┤
│  角色定位：身先士卒，冲锋陷阵                                        │
│  核心职能：现场施工、人员组织、进度执行、质量自检                      │
│  关键特征：                                                          │
│  ├── 直接带兵作战（有士兵）                                          │
│  ├── 接受军师指挥（但听令于自己军团长）                              │
│  └── 评价基于执行效率和战斗成果                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 实体映射关系

```
势力（Faction）
└── 建管单位（按地区划分）
    ├── 长沙公司
    ├── 岳阳公司
    ├── ...（14个市州）
    └── 省直属（500kV专属）

城池（City）
└── 变电站（按电压等级区分规模）
    ├── 特高压站（1000kV）= 都城
    ├── 500kV站 = 州府
    ├── 220kV站 = 郡城
    └── 110kV站 = 县城

道路（Route）
└── 输电线路（杆塔序列）
    └── 每基杆塔 = 路标/关卡

军团（Legion）
└── 施工单位
    ├── 军团长 = 项目经理（主将）
    ├── 武将团 = 副经理/总工/技术员/安全员（副将）
    └── 士兵 = 作业人员（有准入资格）
```

## 三、角色系统详解

### 3.1 军师（业主项目经理）

#### 3.1.1 角色定义

| 属性 | 说明 |
|-----|------|
| **所属** | 建管单位（指挥部） |
| **职能** | 项目策划、资源协调、进度督导、质量把控 |
| **直属单位** | 无（不直接带兵） |
| **指挥范围** | 可管辖多个项目，每个项目任命一个军团 |

#### 3.1.2 军师职责

**与系统职责的区分**：

| 职责类型 | 内容 | 来源 |
|---------|-----|------|
| **系统发布** | 年度开工目标、投产目标、项目里程碑 | 由上级/系统排定，不可更改 |
| **军师策划** | 资源预配、军团选择、风险预判 | 军师自主决策 |
| **军师军令** | 进度督导、问题协调、优先级调整 | 军师主动发布 |
| **军师协调** | 跨单位支援、物资到货、设计变更 | 军师的执行动作 |

#### 3.1.3 军令系统

| 军令类型 | 作用 | 效果 |
|---------|-----|------|
| **督战令** | 要求某项目加快进度 | 健康度改善，军师+协调分 |
| **支援令** | 请求其他军师/军团支援 | 解决资源瓶颈 |
| **调整令** | 申请调整非关键节点 | 优化项目节奏 |
| **请示令** | 向上级申请资源/延期 | 应对突发情况 |

#### 3.1.4 军师评价体系

| 维度 | 权重 | 说明 |
|-----|-----|------|
| **项目策划** | 25% | 开工达成率、计划准确性、资源预配 |
| **协调效率** | 25% | 问题解决速度、跨单位协调、变更控制 |
| **投资控制** | 20% | 预算执行、变更成本、结算效率 |
| **整体达成** | 30% | 投产达成率、质量一次投运、安全零事故 |

### 3.2 武将（施工项目经理及团队）

#### 3.2.1 军团编制

```
军团（施工单位项目部）
├── 主将（项目经理）
│   ├── 职责：全面负责，对外接口
│   ├── 权限：军团内武将调配（需军师批准）
│   └── 必要条件：必须配备备选主将才能转出
│
└── 武将团（副将）
    ├── 副经理
    │   └── 职责：协助主将，分管后勤
    ├── 总工程师
    │   └── 职责：技术方案、质量控制
    ├── 技术员
    │   └── 职责：施工图纸、技术交底
    └── 安全员
        └── 职责：安全监督、风险管控
```

#### 3.2.2 武将评价体系（通用）

所有武将（主将+副将）共用同一评价体系：

| 维度 | 权重 | 说明 |
|-----|-----|------|
| **进度执行** | 25% | 节点达成率、工期控制、响应速度 |
| **质量管控** | 25% | 一次通过率、返工率、工艺水平 |
| **安全绩效** | 25% | 零事故天数、风险管控、违章率 |
| **资源效率** | 25% | 人员效率、资源利用率、成本控制 |

**评级标准**：
- S级：95分以上
- A级：85-94分
- B级：75-84分
- C级：60-74分
- D级：60分以下

### 3.3 士兵（作业人员）

#### 3.3.1 士兵属性

| 属性 | 说明 |
|-----|------|
| **准入资格** | 必须具备相应作业资质 |
| **技能类型** | 高空/电气/机械/土建等 |
| **经验值** | 参与项目的累积经验 |
| **流动记录** | 军团间流动、流入、流出 |

#### 3.3.2 人员流动机制

| 流动类型 | 说明 | 可视化 |
|---------|-----|--------|
| **军团间流动** | A单位人员支援B单位 | 粒子动画（势力色混合） |
| **流入** | 新准入人员加入 | 粒子动画（绿色，从边缘流入） |
| **流出** | 人员退休/调离/资格吊销 | 粒子动画（灰色，向边缘流出） |
| **武将流动** | 项目经理/副将调动 | 系统通知+审批流程 |

### 3.4 人员转移规则

#### 3.4.1 主将转移规则

| 规则 | 说明 |
|-----|------|
| **必须有继任者** | 主将转出必须指定1名副将晋升为备选主将 |
| **需军师批准** | 所有主将转移必须经过军师审批 |
| **可降级转移** | 主将可转任其他项目的副将（如总工、安全员） |
| **战绩保留** | 历史战绩永久保留，不受转移影响 |

#### 3.4.2 副将转移规则

| 规则 | 说明 |
|-----|------|
| **军团内调动** | 可在同一军团内调整角色 |
| **跨军团调动** | 可调任其他军团的副将职位 |
| **晋升主将** | 原主将转出时，备选主将自动晋升 |
| **无需军师批** | 军团内部调动由主将决定，跨军团需军师协调 |

## 四、项目生命周期与结算机制

### 4.1 项目生命周期（三态模型）

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   工程前期   │ → │    在建     │ → │   已投产    │
│  (未开工)   │    │  (建设中)   │    │  (完结)     │
└─────────────┘    └─────────────┘    └─────────────┘
       │                  │                  │
       ▼                  ▼                  ▼
   系统发布开工目标    系统发布投产目标      赛季结算时
   军师策划推进        军师军令督促进度      累计战绩结算
```

#### 4.1.1 状态说明

| 状态 | 定义 | 关键动作 |
|-----|------|---------|
| **工程前期** | 已立项但未正式开工 | 军师策划、资源预配 |
| **在建** | 已开工至投产前的全过程 | 武将施工、军师督导 |
| **已投产** | 已完成投产移交 | 战绩结算、归档 |

### 4.2 结算机制

#### 4.2.1 结算原则

| 事件类型 | 结算时机 | 说明 |
|---------|---------|------|
| **加分事件** | 延迟结算（投产时统一发放） | 质量优秀、提前节点等正向激励 |
| **扣分事件** | 实时结算（立即生效） | 安全事故、质量问题等负向约束 |

#### 4.2.2 结算节点

| 节点 | 触发条件 | 结算内容 |
|-----|---------|---------|
| **开工结算** | 项目正式开工时 | 军师获得开工达成战绩 |
| **过程评价** | 季度/月度 | 仅反馈，不结算（加分累计到投产） |
| **投产结算** | 项目投产时 | 军师+武将获得基础战绩+累计加分 |

#### 4.2.3 结算事件示例

```
投产结算事件 - 长沙西500kV变电站：

基础战绩：
├─ 军师李规划：开工结算 +20分
├─ 军师李规划：投产结算 +30分
├─ 主将张建设：项目完成 +25分
└─ 副将李技术：项目完成 +20分

延迟结算发放（加分池）：
├─ 军师李规划：+12分（质量优秀+5，提前节点+5，协调高效+2）
├─ 主将张建设：+8分（零违章+3，提前基础节点+5）
└─ 副将李技术：+5分（零技术问题）

实时结算已扣：
└─ 无

本赛季累计：+110分
```

#### 4.2.4 扣分事件示例

```
实时扣分事件 - 衡阳线路工程：

事件：安全违章（高处作业未系安全带）
时间：2025-04-10 14:32

扣分：
├─ 军师王指挥：-5分（安全管理责任）
├─ 主将刘施工：-10分（现场管理责任）
└─ 副将刘安全：-15分（专职安全责任）

影响：
└─ 该项目健康度调整为"🟡预警"
```

## 五、赛季系统

### 5.1 赛季结构

| 赛季属性 | 定义 |
|---------|------|
| **赛季周期** | 自然年度（2025赛季、2026赛季...） |
| **目标发布** | 年初系统发布开工目标、投产目标 |
| **过程跟踪** | 全年实时跟踪进度和健康度 |
| **年底结算** | 统计所有已完结项目的战绩 |

### 5.2 赛季目标

| 目标类型 | 内容 | 来源 |
|---------|------|------|
| **开工目标** | 计划新开工X个项目 | 系统排定 |
| **投产目标** | 计划投产Y个项目（容量Z MVA） | 系统排定 |
| **过程目标** | 安全/质量/进度控制指标 | 系统发布 |

### 5.3 赛季结算

#### 5.3.1 结算范围

| 项目状态 | 结算方式 |
|---------|---------|
| **已投产** | 完整结算（开工战绩+投产战绩+累计加分） |
| **在建** | 仅结算开工战绩（投产战绩和加分留待下一年度） |
| **工程前期** | 不计入本赛季 |

#### 5.3.2 建管单位赛季总结

```
军师赛季总结示例 - 长沙供电公司建设部 2025赛季：

开工战绩：
├─ 目标：8个项目
├─ 实际：8个项目
├─ 达成率：100%
└─ 得分：+40分

投产战绩：
├─ 目标：6个项目（容量1800MVA）
├─ 实际：5个项目（容量1500MVA）
├─ 达成率：83%
└─ 得分：+33分

过程评价：
├─ 安全绩效：+20分（零事故）
├─ 质量绩效：+15分（一次投运率100%）
└─ 投资控制：+5分（节约3%）

赛季总得分：113分 / 目标120分
达成率：94%
评级：A级
排名：市州第2名（共14个）
```

#### 5.3.3 施工单位赛季总结

```
军团赛季总结示例 - 湖南送变电 2025赛季：

参与项目：12个
├─ 主将项目：8个（担任项目经理）
├─ 副将项目：4个（担任总工/安全员等）
└─ 完结项目：10个

平均项目评级：A级（87分）

武将战绩：
├─ 主将张建设：+125分（3个项目）
├─ 主将王施工：+98分（2个项目）
├─ 副将李技术：+76分（5个项目）
└─ ...

安全记录：
└─ 零事故项目比例：90%（9/10）

军师评价：
└─ 平均配合度：4.5/5.0
```
