#### 1.3 建设进度：多源融合模型

你提到进度不能简单线性计算，因为有多个信息源。我建议设计一个**"工程健康度"**综合指标：

```
工程健康度 = f(质量验评, 转序进度, 作业票活跃度, 物资到货, ...)

信息源权重（示例）：
┌─────────────────┬────────┬─────────────────────────────┐
│ 信息源          │ 权重   │ 说明                        │
├─────────────────┼────────┼─────────────────────────────┤
│ 质量验评表      │ 30%    │ 质量关卡通过情况            │
│ 单位工程转序    │ 25%    │ 阶段性里程碑完成            │
│ 作业票密度      │ 20%    │ 近期作业活跃度（人力投入）   │
│ 物资到货        │ 15%    │ 设备材料到场情况            │
│ 设计变更        │ 10%    │ 变更频次（负向指标）         │
└─────────────────┴────────┴─────────────────────────────┘

可视化方式：
- 不要显示"完成68%"这种精确数字
- 改用状态标签：筹备中/基础施工/组塔阶段/架线阶段/验收中/投产
- 加上"健康度条"：红(滞后)/黄(预警)/绿(正常)/蓝(超前)
```

### 二、工程骨架可视化方案

#### 2.1 地图分层设计

```
Layer 0: 底图（湖南省行政区划）
Layer 1: 势力范围（建管单位辖区，半透明色块）
Layer 2: 城池（变电站）
         ├── 图标：按电压等级区分大小
         ├── 状态：在建(闪烁)/已投产(常亮)/规划中(灰)
         └── 交互：hover显示基本信息，点击打开详情面板
Layer 3: 道路（输电线路）
         ├── 线条：连接变电站
         ├── 颜色：按建设状态（红/黄/绿/蓝）
         └── 动画：建设中的线路有流动效果
Layer 4: 路标（杆塔）
         ├── 小圆点沿线路分布
         ├── 颜色：已建(绿)/在建(黄)/待建(灰)
         └── 密度：缩放后显示
Layer 5: 军团位置（当前作业点）
         └── 使用现有站班会数据，叠加显示
```
