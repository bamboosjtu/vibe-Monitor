# 当前状态

## 项目定位

`vibe-Monitor` 是数字沙盘消费端。当前它的主职责是消费 `vibe-DataCollectorHub` 的 sandbox API，并把 DataHub 返回的稳定 DTO 展示为地图、时间轴、筛选、统计和详情。

主链路：

```text
vibe-downloader
  -> vibe-DataCollectorHub
  -> GET /api/v1/sandbox/*
  -> vibe-Monitor frontend
```

## 当前 MVP 范围

显示：

- `daily_meeting` -> work points
- `tower` -> towers
- `station` -> stations

暂不显示：

- `line_section`
- `year_progress`

## 数据源模式

| 模式 | 说明 | 当前用途 |
| --- | --- | --- |
| `datahub` | 调用 DataHub sandbox API | 主链路 |
| `local` | 读取本地 JSON | fallback / demo |
| `api` / `legacy-api` | 调旧 backend API | 兼容旧流程 |

DataHub mode 需要：

```text
VITE_DATA_SOURCE=datahub
VITE_DATAHUB_BASE_URL=http://localhost:8000
```

## DataHub API

当前使用：

```text
GET /api/v1/sandbox/dates
GET /api/v1/sandbox/map/summary?date=YYYY-MM-DD
GET /api/v1/sandbox/map/skeleton
```

前端文件入口：

```text
frontend/api/config.ts
frontend/api/datahubApi.ts
frontend/api/adapter.ts
frontend/lib/useDataLoader.ts
frontend/store/index.ts
```

## DTO 边界

Monitor 不依赖 DCP raw 字段。

禁止在 Monitor 业务组件中直接使用：

- `toolBoxTalkLongitude`
- `toolBoxTalkLatitude`
- `currentConstrHeadcount`
- `reAssessmentRiskLevel`
- `currentConstructionStatus`
- `longitudeEdit`
- `latitudeEdit`
- `singleProjectCode`

这些字段只能在 DataHub normalizer 或 DataHub adapter 入参类型中出现。Monitor 内部 view model 应使用稳定字段，例如：

- `id`
- `name` / `projectName`
- `longitude`
- `latitude`
- `riskLevel`
- `workStatus`
- `personCount`

## backend 状态

`backend/` 保留。

当前第一版 DataHub 联调不依赖 backend；未来 backend 可以作为 BFF / 应用处理层，处理 DataHub 已治理数据之上的消费侧逻辑。

backend 不应承担：

- DCP 采集。
- SourceEvent ingestion。
- raw_events 存储。
- canonical normalizer。
- DataHub scheduler / checkpoint。

## 与基建 online 的关系

当前数字沙盘是基建 online 的底座能力之一。

`doc/exp/基建online策划方案v1.2.md` 是长期目标主文档；当前代码不应一次性实现其中的赛季、军令、工单、战绩、履历等机制。这些能力应在 DataHub 主链路稳定后，通过 backend / BFF / 应用服务逐步承接。
