# 开发指南

**更新时间**: 2026-05-03  
**适用状态**: 当前代码已进入 M1 Round2，前后端分离，前端默认 local 静态 JSON，后端作为数据底座和 API 兼容层。

---

## 1. 接手前必读

1. 先读 [doc/项目交接说明.md](doc/项目交接说明.md)。
2. 前端默认不依赖后端，`cd frontend && npm run dev` 即可演示。
3. 后端依赖由 uv 管理，不再使用 `requirements.txt`。
4. 仓库根目录 `data/` 是正式源数据目录，`backend/data/` 只存 SQLite 数据库。
5. 当前实现仍有调试遗留：`MapContainer.tsx` 中有 M1R2 debug overlay 和 console 日志。

## 3. 当前运行方式

### 前端

```bash
cd frontend
npm install
npm run dev
```

前端默认 `dataSource: 'local'`。页面启动后：

1. 读取 `/data/daily_meeting/index.json`
2. 取最后一个日期作为默认日期
3. 请求 `/data/daily_meeting/meeting_YYYYMMDD_all.json`
4. 解析、过滤、归一化后写入 Zustand store

`frontend/vite.config.ts` 中的 `serve-repo-data` 插件会在开发/预览环境把仓库根目录 `data/` 映射到浏览器 `/data/`。

### 后端

```bash
cd backend
uv sync --dev
uv run python scripts/db/align_db.py
uv run python run.py
```

后端默认 `http://localhost:8000`。当前前端 API 客户端默认 `http://localhost:8001`，切 API 模式前需要统一端口。

## 4. 开发边界

优先级：

1. 数据口径正确
2. 运行路径清晰
3. 前后端边界明确
4. 测试可重复
5. UI 优化最后处理

当前不要默默改变的口径：

- 时间轴按天，不按小时
- 作业点统计基于合法坐标记录
- 地图标记数量应与当日作业点总数一致
- 风险 unknown 在 UI 上展示为“无风险”
- API 模式是兼容能力，local 模式仍是默认演示路径

---

## 5. 数据规则

### 坐标

过滤不上图、不参与统计：

- 经纬度为空
- 经纬度非数字
- 经纬度为 0
- 超出湖南范围：经度 108.6~114.3，纬度 24.6~30.2

### 人数

- 空值、非数字、负数视为 0
- 大于 300 时，地图半径映射按 300 截断，详情展示保留原值

### 状态与风险

- 作业状态未知值归为 `unknown`
- 风险等级支持 1-4 与 `unknown`
- `unknown` 风险当前 UI 展示为“无风险”

## 6. 代码组织

### 前端

- `frontend/lib/dateDataLoader.ts`: manifest 与按日期 JSON 加载
- `frontend/lib/dataParser.ts`: 原始数据清洗与归一化
- `frontend/lib/filter.ts`: 筛选逻辑
- `frontend/lib/stats.ts`: 统计逻辑
- `frontend/store/index.ts`: 全局状态
- `frontend/features/map/MapContainer.tsx`: MapLibre 图层与点击交互
- `frontend/features/timeline/Timeline.tsx`: 日期切换和自动播放

### 后端

- `backend/app/api/`: API 路由
- `backend/app/importers/`: 导入器
- `backend/app/models/m0_models.py`: 数据库模型
- `backend/scripts/db/`: 数据库维护脚本
- `backend/scripts/imports/`: 数据导入与聚合脚本
- `backend/scripts/diagnostics/`: 人工诊断脚本
- `backend/tests/`: 未来 pytest 测试目录
