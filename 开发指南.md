# 开发指南

本文只描述当前 `vibe-Monitor` 的开发入口。旧 M0/M1 过程说明已清理。

## 当前开发主线

当前主线是 Monitor backend mode：

```text
DataHub Domain API -> Monitor backend -> frontend/api/monitorBackendApi.ts -> frontend/api/adapter.ts -> store -> map/filter/detail/timeline
```

开发时必须遵守：

- 不在前端读取 DCP raw 字段。
- 不在前端解析 Envelope / raw_event 存储结构。
- 不在前端直接访问 DataHub raw layer、downloader 或旧 sandbox API。
- DataHub DTO 到 Monitor view model 的转换放在 `frontend/api/adapter.ts` 或相邻 adapter 文件。
- local JSON mode 保留作为 fallback，不要移除。
- 前端生产路径只允许 `monitor_backend` 和 `local`。

## 目录重点

```text
frontend/
  api/
    config.ts          # 数据源配置
    datahubApi.ts      # 共享 DTO 类型定义
    adapter.ts         # DataHub/local/API DTO -> Monitor view model
  lib/
    useDataLoader.ts   # 首屏和日期切换加载入口
  store/
    index.ts           # 全局状态与数据加载 action
  features/
    map/               # MapLibre 地图
    filter/            # 筛选
    detail/            # 详情
    stats/             # 统计
    timeline/          # 时间轴
  constants/           # 常量定义
  scripts/             # 构建和数据处理脚本
  test/                # 测试工具

backend/
  app/                 # Monitor backend BFF API
  scripts/             # backend 诊断脚本
  tests/               # backend 测试目录
```

## 常见约束

- Monitor backend mode 下，时间轴日期来自 `/api/map/dates`。
- 日期切换只重新请求 `/api/map/summary?date=...`。
- skeleton 可缓存，不需要每次日期切换重拉。
- Monitor DTO 不应包含或透传 `raw`。
- `line_section` 和 `year_progress` 通过 Domain API 读模型进入当前 Monitor 主链路。
